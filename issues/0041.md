## Kontext
Im Karten-Detail sind Anhänge aktuell Download-Links (nur Bilder zeigen eine kleine Inline-Miniatur). Der Nutzer will beim Klick auf einen Anhang eine Vorschau. Umfang: Bilder und PDF inline im Overlay, andere Typen weiterhin Download.

## Aufgabe
- `frontend/src/components/CardDetailModal.tsx` (Anhänge-Bereich): Klick auf einen Anhang öffnet eine **Lightbox/Overlay-Dialog**:
  - `image/*` → Bild groß im Overlay,
  - `application/pdf` → PDF im Overlay (z. B. `<iframe>`/`<object>` auf die Blob-URL),
  - sonst → Download (bestehendes Verhalten).
- Vorschau lädt den Blob über `attachmentsApi.fetchBlob` und erzeugt eine Object-URL (wie bei der bestehenden Bild-Miniatur); URL beim Schließen/Unmount via `URL.revokeObjectURL` freigeben (kein Leak).
- Der Download-Weg (`/api/attachments/{id}`, Content-Disposition: attachment) bleibt als Fallback/Explizit-Download bestehen — die Vorschau nutzt ausschließlich die clientseitige Blob-URL (kein Inline-Render vom Server, Anti-XSS bleibt).
- Eine ggf. neue kleine Komponente `AttachmentPreview`/Dialog; Tastatur-schließbar (ESC), Klick auf Backdrop schließt.
- Tests: Klick auf einen Bild-Anhang öffnet das Overlay mit `<img>`; PDF öffnet das PDF-Overlay; unbekannter Typ löst keinen Overlay aus (Download-Link bleibt).

## Akzeptanzkriterium
- `npm test` + `npm run build` grün.
- Klick auf einen Bild- oder PDF-Anhang zeigt eine Vorschau im Overlay; andere Anhänge werden heruntergeladen; keine Object-URL-Leaks.
