## Kontext
Für das Dogfooding (D1) sollen das Stellwerk-`board.mjs` und die bestehende `tbx.mjs`-CLI das eigene kanban-kit-Board als Issue-Tracker treiben. Beide Clients senden ausschließlich den PAT im Header `X-Kanban-Token` — **keine** Board-ID. Der Server muss daher wissen, welches Projekt/Board zu einem Token gehört. Heute ist ein `AccessToken` nur nutzer-gebunden (`user_id`), ohne Projekt/Board. Damit ein Token wie ein GitHub-Fine-grained-PAT genau ein Board adressiert (und der Zugriff darauf beschränkt bleibt), muss das Token an Projekt + Board gebunden werden.

## Aufgabe
- Flyway `V3__kanban_token_board_binding.sql`: Spalten `project_id BIGINT NULL`, `board_id BIGINT NULL` an `kanban_access_token` (nullable — Bestandstokens bleiben ungebunden).
- `AccessToken` (Domain-Record): Felder `projectId`, `boardId` ergänzen; `KanbanAccessTokenEntity` + Repository-Adapter nachziehen.
- `AccessTokenService.create(userId, name, projectId, boardId)`: bei gesetzter Bindung validieren, dass das Board zum Projekt gehört und der Nutzer Mitglied mit ausreichender Rolle ist (PermissionChecker). `projectId`/`boardId` optional (ungebundenes Token bleibt möglich).
- Auflösung des Bindings für nachgelagerte Endpunkte bereitstellen: neue Methode `resolveBinding(plaintext) -> Optional<KanbanPrincipal(userId, tokenId, projectId, boardId)>`; `PatAuthenticationFilter` hängt das Binding an die Authentication-`details`, sodass die Kanban-Endpunkte (#45) das gebundene Board ohne zweiten Lookup kennen.
- `AccessTokenController.create` + Request-DTO um optionale `projectId`/`boardId` erweitern; `AccessTokenView`/List die Bindung mit ausgeben.

## Akzeptanzkriterium
- Migration läuft (Flyway) sauber durch; `mvn test` grün.
- IT: gebundenes Token → `resolveBinding` liefert projectId+boardId; ungebundenes → leeres Binding, aber weiterhin gültige userId.
- IT: Binding auf ein Board, in dem der Nutzer kein Mitglied ist, wird bei `create` mit 403 abgelehnt; Board, das nicht zum Projekt gehört, mit 400.
- Bestehende PAT-Auth (nur userId) bleibt unverändert funktionsfähig (Regression grün).
