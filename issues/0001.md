## Kontext
manban startet als frisches Repo, herausgelöst aus der Toolbox. Ohne lauffähiges Gerüst kann kein Feature getestet werden.

## Aufgabe
- Maven-Projekt Java 21 / Spring Boot 3.5 (`org.mwolff.manban`), hexagonale Paketstruktur (`domain/application/web/infrastructure` je Modul).
- React 18 / Vite 5 / TypeScript strict / MUI 6 unter `frontend/`.
- Multi-Stage-Dockerfile (Node → Maven → JRE).
- `docker-compose.yml` mit `manban-api`, `postgres`, `minio`, `caddy` + `Caddyfile`.
- `.env.example`.
- CI (Build + Test).
- `frontend-maven-plugin` fürs gebündelte Frontend (wie in Toolbox).
- Postgres-Treiber + Flyway eingebunden.

## Akzeptanzkriterium
- `mvn verify` grün (leeres Skelett).
- `cd frontend && npm run build` grün.
- `docker compose up` bringt alle vier Container hoch.
- `/` liefert die React-App über Caddy mit TLS-Redirect.
- On-Prem-tauglich: lauffähig ohne weiteren Repo-Kontext.
