## Kontext
[CLAUDE-react.md](CLAUDE-react.md) → Routing: „URL-Parameter validieren oder defensiv interpretieren (`Number.parseInt(id, 10)` + Range-Check, nicht naked `+id`)." Mehrere Seiten machen `const id = Number(boardId)` ohne jede Prüfung. `Number('abc')` → `NaN`, `Number('')` → `0`; der ungeprüfte Wert geht direkt in API-Aufrufe.

## Aufgabe
- In folgenden Dateien den URL-Param robust parsen und guarden:
  - `frontend/src/pages/BoardListPage.tsx:69`
  - `frontend/src/pages/BoardPage.tsx:19`
  - `frontend/src/pages/EpicsPage.tsx:31`
  - `frontend/src/pages/ProjectBoardsPage.tsx:23`
  - `frontend/src/pages/ProjectMembersPage.tsx:30`
- Muster: `const id = Number.parseInt(param ?? '', 10)` mit Guard `Number.isInteger(id) && id > 0`; bei ungültigem Wert einen definierten Fehler-/Redirect-Zustand rendern (nicht blind an die API geben).
- Tests: Fall „ungültiger Param" ergänzen (zeigt Fehlerzustand statt Fetch mit NaN/0).

## Akzeptanzkriterium
- Kein `Number(param)` ohne anschließenden Integer/Range-Check mehr in den genannten Seiten.
- Test für ungültigen Param grün; `tsc --noEmit` sauber, `vitest run` grün.
