## Kontext
Die aktuelle `/roles`-Seite (`RolesPage.tsx`) zeigt eine statische Text-Tabelle. Sie soll durch ein **Checkbox-Grid** ersetzt werden: Spalten = einzelne granulare Rechte (je Operation eine Spalte), Zeilen = Rollen. Für die vier eingebauten Rollen sind die Haken **disabled** (read-only). Struktur so, dass 2.0 nur „Rolle als Zeile anhängen + Haken enablen" ist.

## Aufgabe
- `RolesPage.tsx`: Matrix aus `GET /api/roles/matrix` laden (neue `api/roles.ts`), nicht mehr hart kodieren.
- Grid rendern: Spalten nach Ressource gruppiert (Board · Epic · Ticket · Kommentar · Anhang · Move · Verwaltung) mit Unterspalten je Operation (C/R/U/D); Zeilen = Rollen; Zelle = Checkbox `checked` = Grant, `disabled` (eingebaute Rollen).
- Horizontal scrollbar bei vielen Spalten; responsive.

## Akzeptanzkriterium
- Grid entspricht exakt der Backend-Matrix (Grants als gesetzte, disabled Haken).
- Alle Rollen als Zeilen, alle Permissions als (gruppierte) Spalten sichtbar.
- Erweiterbar dokumentiert (2.0: weitere Rollen-Zeile, enabled Haken).
- Vitest-Test rendert Grid gegen ein gemocktes Matrix-Response; Build grün.

## Abhängigkeiten
#51 (Matrix-Endpunkt).
