## Kontext
Die aktuelle App-Version soll im laufenden Betrieb sichtbar sein, ohne in Dateien nachsehen zu müssen — konkret im Header des Kanban-Boards. Der App-weite Header (`AppBar`/`Toolbar` in [AppShell.tsx](frontend/src/components/AppShell.tsx:287)) zeigt heute nur Logo und Produktname "kanban-kit"; er ist fix positioniert und auf jeder Seite inkl. Board-Ansicht sichtbar — damit der richtige, einzige Ort für die Anzeige. Voraussetzung ist eine synchron gepflegte `frontend/package.json`-Version, die #0105 liefert.

## Aufgabe
- `frontend/vite.config.ts`: `package.json`-Version zur Build-Zeit einlesen und per `define: { __APP_VERSION__: JSON.stringify(pkg.version) }` in den Build injizieren.
- `frontend/src/vite-env.d.ts`: `declare const __APP_VERSION__: string` ergänzen.
- `frontend/src/components/AppShell.tsx`: neben dem Produktnamen (Zeile ~289–294) eine kleine `Typography` mit `v{__APP_VERSION__}` ergänzen — dezente Darstellung (z. B. `variant="caption"`, reduzierte Opacity), kein visuelles Gewicht wie der Produktname, sichtbar auf jeder Route (nicht nur `/boards/:id`).
- Test in `AppShell.test.tsx` ergänzen.

## Akzeptanzkriterium
- `npm run build` erfolgreich; das `dist/`-Bundle enthält die zum Build-Zeitpunkt aktuelle `package.json`-Version als konkreten String (kein Platzhalter/`undefined`).
- `AppShell.test.tsx`: neuer Test prüft, dass `v` + die (im Test gemockte) Version im Header gerendert wird.
- Manueller Check im Dev-Server: Board-Seite geöffnet → Version ist im oberen Header sichtbar; Wechsel auf eine andere Route (z. B. Projekte) → weiterhin sichtbar.
- `npm run build` + `npm run lint` + `npm test` grün.

## Abhängigkeiten
#0105 (Versionsschema muss zuerst stehen, damit `package.json` die tatsächliche Betriebsversion trägt statt einer statischen `0.1.0`).
