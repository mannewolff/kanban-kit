## Kontext
Bei nur einem Projekt bzw. nur einem Board sind die Auswahl-Listen überflüssige Zwischenschritte. Der Nutzer will bei genau einem Element direkt durchgeroutet werden.

## Aufgabe
- `frontend/src/pages/ProjectsPage.tsx`: nach dem Laden — wenn **genau ein** Projekt existiert, per `navigate(`/projects/${id}`, { replace: true })` direkt zur Boardauswahl.
- `frontend/src/pages/ProjectBoardsPage.tsx`: wenn **genau ein** Board existiert, per `navigate(`/boards/${id}`, { replace: true })` direkt aufs Board.
- **Loop-/Sackgassen-Schutz:** Auto-Routing darf den Nutzer nicht einsperren (sonst käme man nie zur Liste zurück, um ein 2. Projekt/Board anzulegen). Die Rücksprung-Links ("← Projekte" bzw. "← Boards") setzen ein Navigations-Signal (z. B. `state: { stay: true }` oder Query `?stay=1`), das das Auto-Routing auf der Zielseite unterdrückt. Ebenso soll die Sidebar-Navigation zur Liste nicht sofort wieder wegrouten.
- Auto-Routing nur beim regulären Aufruf, nicht während des Ladens (erst nach vorhandener Antwort entscheiden; kein Flackern/Doppel-Redirect).
- Tests: bei 1 Projekt Redirect auf /projects/:id; bei mehreren keine Umleitung; bei 1 Board Redirect auf /boards/:id; mit `stay`-Signal keine Umleitung (Liste bleibt sichtbar).

## Akzeptanzkriterium
- `npm test` + `npm run build` grün.
- Login mit genau 1 Projekt landet direkt in der Boardauswahl; genau 1 Board landet direkt auf dem Board; über den Zurück-/Sidebar-Weg bleibt die Liste erreichbar (um weitere anzulegen), ohne sofortiges Zurückrouten.
