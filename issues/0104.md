## Kontext
Mit der umgezogenen CLI (#0103) und dem `toolbox`-Adapter in `board.mjs` (bereits vorhanden, #0013) kann kanban-kit sein eigenes Board als Issue-Tracker treiben. Dieser Durchstich verifiziert die Kette Ende-zu-Ende und dokumentiert sie — das eigentliche „Dogfooding ausprobieren".

## Aufgabe
- Board-gebundenes Token via `POST /api/access-tokens` (Session-Cookie) für ein Projekt/Board erzeugen.
- `node cli/tbx.mjs auth login --host https://localhost --token tk_…`.
- `.claude/workflow.config.json`: `issueTracker` von `local` → `toolbox` (+ `toolbox.host`) umstellen.
- Smoke-Test: `node .claude/kit/board.mjs issue list` / `issue create` / `issue move <n> in_progress` → Karten erscheinen live im Web-Board.
- `docs/dogfooding.md` aktualisieren: CLI-Aufruf `node cli/tbx.mjs …` statt `tbx …`; Herkunft der CLI jetzt im Repo.

## Akzeptanzkriterium
- Eine via `board.mjs issue create` angelegte Karte erscheint im Web-Board; `issue move` verschiebt die Spalte korrekt (`backlog↔Backlog` … `done↔Done`).
- `docs/dogfooding.md` beschreibt den Ablauf mit `cli/tbx.mjs`.
