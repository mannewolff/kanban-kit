## Kontext
Die Epics-Ansicht soll wie in der Vorlage aussehen: je Epic eine volle-Breite-Zeile mit Epic-Badge, Titel und **Fortschrittsbalken** samt "X/Y Stories fertig"-Label. Außerdem soll es dort einen **„Neues Epic"**-Knopf geben (nicht „Neues Item"). Klick auf ein Epic öffnet weiterhin das Detail (Entscheidung bestätigt).

## Aufgabe
- `frontend/src/pages/EpicsPage.tsx`:
  - Zeilen-Layout an die Vorlage angleichen: Epic-Badge + Titel links, rechts/darunter ein Fortschrittsbalken (`done/total`) mit Label "${done}/${total} Stories fertig"; volle Breite; Hover.
  - **„Neues Epic"-Button** oben rechts, der den Anlege-Dialog **direkt im Epic-Modus** öffnet (Titel + Beschreibung + optional Kürzel; kein Typ-Umschalter/keine Spaltenwahl). Nach Anlegen Epics neu laden.
  - Klick auf eine Epic-Zeile öffnet weiterhin das Detail-Modal (Kürzel + Kinderliste) — unverändert.
- Der Anlege-Dialog kann der bestehende `NewCardModal` mit vorbelegtem Typ=EPIC sein (Typ-Auswahl dann ausgeblendet) oder ein schlanker Epic-Dialog — je nachdem, was sauberer ist.
- Tests: Epics-Liste zeigt Fortschritt/Label; „Neues Epic"-Button öffnet den Epic-Anlege-Dialog und ruft `epicsApi.create`; Klick auf ein Epic öffnet das Detail.

## Akzeptanzkriterium
- `npm test` + `npm run build` grün.
- Epics erscheinen als volle-Breite-Zeilen mit Fortschrittsbalken + "X/Y Stories fertig"; ein „Neues Epic"-Button legt ein Epic an; Klick öffnet das Epic-Detail.
