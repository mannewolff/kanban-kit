## Kontext
Self-Service-Signup ist die Eintrittstür. SMTP nötig (der Punkt, der in der Toolbox #262 blockierte).

## Aufgabe
- `app_user` anlegen, Passwort-Hashing Argon2id (Spring Security `Argon2PasswordEncoder`).
- `POST /api/auth/register` (E-Mail + Passwort + Name), Verifikations-Token in `email_verification_token`.
- Versand per SMTP (`JavaMailSender`, konfigurierbar, in Dev Mock/Log).
- `GET /api/auth/verify?token=…`.
- Neu registrierte User: `email_verified=false`, kein Login bis verifiziert.

## Akzeptanzkriterium
- Registrierung legt User an, sendet Verifikations-Mail (Dev: Log).
- Verify setzt `email_verified=true`.
- Doppelte E-Mail → 409.
- Tests für Happy Path + abgelaufener/ungültiger Token.
- `mvn verify` grün.
