## Kontext
Admins brauchen in der Nutzerverwaltung eine Sicht auf offene Freigaben und einen Button zum Freigeben. Das Backend liefert Approval-Feld und Endpunkt aus #0097.

## Aufgabe
- `frontend/src/api/admin.ts`: `AdminUser` um `approvedAt: string | null` erweitern; `approve(id)` → `POST /api/admin/users/{id}/approve`.
- `frontend/src/pages/AdminPage.tsx`: Status-Spalte („Freigegeben" / „Wartet auf Freigabe"), Aktion „Freigeben" (nur bei pending sichtbar), optimistisches Update + Fehlerbehandlung analog zum bestehenden Rollen-Toggle.
- Vitest-Tests.

## Akzeptanzkriterium
- Pending-User zeigt „Wartet auf Freigabe" + „Freigeben"-Button; nach Klick erscheint er als „Freigegeben", Button verschwindet.
- `npm run build` + `npm run lint` + `npm test` grün.
