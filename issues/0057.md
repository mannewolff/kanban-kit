## Kontext
`frontend/src/lib/statusColors.ts` ist die aus der Toolbox portierte **Atlassian-/Trello-Palette** (`#deebff`, `#fffae6`, `#ffedeb`, `#e3fcef`, Spalten `#ebecf0`, Modal-Grau `#344563`). Diese kräftigen, bunten Flächen sind der Hauptgrund für das generische „Board-Gefühl" und stehen im Widerspruch zur feinen, hellen Teal-Marke. Entscheidung: Spalten-Header weiß + Eyebrow-Label + dezenter Punkt (statt farbiger Vollfläche).

## Aufgabe
- `frontend/src/lib/statusColors.ts`:
  - Statusfarben auf die Teal-/Marken-Familie entsättigen (z. B. Backlog `#5BABB5`, Ready `#2F8C97`, Done teal-grün, In Progress/Review gedämpftes Amber/Coral) — vor allem die `dot`-Farbe bleibt semantisch, `bg` wird ruhig/hell.
  - `COLUMN_SURFACE_BG` von `#ebecf0` auf weiß bzw. `#F6FAFB`.
  - Modal-Chrome-Konstanten (`MODAL_BORDER`, `MODAL_HEADER_BG`, `MODAL_TEXT_PRIMARY`, `MODAL_TEXT_SECONDARY`) auf Marken-Töne (`#D8ECEE`, `#F6FAFB`, `#243539`, `#5F7A7F`).
- `frontend/src/components/BoardView.tsx`: Spalten-Header nicht mehr als farbige Vollfläche, sondern **weiß + Eyebrow-Label** (klein, gesperrt, Großbuchstaben, `#5BABB5`) **+ dezenter Farbpunkt** (`colors.dot`) + Zähler-Pill (`#F6FAFB`/`#D8ECEE`).
- `frontend/src/lib/statusColors.test.ts`: Erwartungswerte auf die neuen Farben nachziehen.

## Akzeptanzkriterium
- Dev-Server: Board-Spalten-Header sind hell/ruhig mit Eyebrow-Label + Punkt; keine bunten Atlassian-Flächen mehr; Karten-Detail-Modal in Marken-Tönen.
- `vitest run` grün (inkl. angepasstem `statusColors.test.ts`), `tsc --noEmit` sauber, `npm run build` ok.
