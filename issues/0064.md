## Kontext
[CLAUDE-react.md](CLAUDE-react.md) → MUI/Styling: „Farben … über das Theme, nicht hartcodiert." Mehrere Komponenten setzen Roh-Hex direkt im `sx`, obwohl `theme.ts` exakt dieselben Werte bereits als Token führt — die Token werden also umgangen und dupliziert.

## Aufgabe
- Hartcodierte Hex-Werte im `sx` auf Theme-Referenzen umstellen:
  - `frontend/src/components/BoardView.tsx:197,199,202,230,231,235`
  - `frontend/src/components/AppShell.tsx:266,328`
  - `frontend/src/components/CardDetailModal.tsx:57,58`
- Mapping (Beispiele): `'1px solid #D8ECEE'` → `divider`; `color: '#5F7A7F'` → `text.secondary`; `borderColor: '#5BABB5'` → `primary.light`; `borderLeft: '4px solid #2F8C97'` → `primary.main`.
- Die in `statusColors.ts` bereits zentral definierten Chrome-Konstanten (`MODAL_BORDER` etc.) nicht als Literal duplizieren, sondern referenzieren; Status-Palette möglichst ins Theme heben.

## Akzeptanzkriterium
- Keine Roh-Hex-Literale mehr in `sx`/`style` der genannten Komponenten (Farben kommen aus Theme-Token bzw. zentralen Konstanten).
- Visuell unverändert (Dev-Server-Sichtprüfung), `tsc --noEmit` sauber, `vitest run` grün, `npm run build` ok.
