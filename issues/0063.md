## Kontext
[CLAUDE-react.md](CLAUDE-react.md) → Performance-Budget: „Route-Level Lazy Loading ist Pflicht für alle Top-Level-Routen in `App.tsx` (via `React.lazy` + `Suspense`). Kein direktes Import einer Page-Komponente in `App.tsx` ohne `lazy()`." Aktuell werden in `frontend/src/App.tsx:2-17` alle 15 Page-Komponenten statisch importiert und ohne `Suspense` gerendert — kein einziges `React.lazy`. Damit landet der gesamte Seiten-Code im initialen Bundle.

## Aufgabe
- `frontend/src/App.tsx`: jede Routen-Page über `const XPage = lazy(() => import('./pages/XPage'))` laden (Named-Exports ggf. via `.then(m => ({ default: m.XPage }))`).
- Die `<Routes>` in ein `<Suspense fallback={…}>` einhüllen (sinnvoller Lade-Fallback, z. B. zentrierter MUI-`CircularProgress`).
- `AppShell` bleibt statisch (Shell), nur die Pages werden lazy.
- Bestehende Tests anpassen; ggf. Test ergänzen, der einen Lade-Fallback zeigt.

## Akzeptanzkriterium
- `App.tsx` importiert keine Page-Komponente mehr direkt ohne `lazy()`; ein `<Suspense>`-Boundary umschließt die Routen.
- `npm run build` erzeugt separate Route-Chunks (in der Vite-Ausgabe sichtbar), alle < 600 kB.
- `tsc --noEmit` sauber, `vitest run` grün, `npm run lint` grün.
