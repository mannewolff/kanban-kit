## Kontext
Freigegebener Plan (2026-07-11): Upgrade von Java 21 auf **Java 25 (LTS)** — lokal, Docker-Build, Runtime-Container, CI. Bewusste Abgrenzung: Spring Boot bleibt auf 3.5.x (Patch-Update ≥ 3.5.6 für offiziellen Java-25-Support); **Spring Boot 4.0 + JUnit 6 sind ein separater Folgeplan**. Keine neuen Sprachfeatures, kein Preview — das Upgrade bleibt mechanisch/verhaltensneutral, damit Fehlschläge zuordenbar sind. Alle Gates (JaCoCo 100 %, PIT 100 %, ArchUnit, Spotless, Checkstyle/PMD/SpotBugs/Error Prone, -Xlint:all -Werror, Enforcer) bleiben unverändert scharf. Zielsystem braucht nichts (JDK steckt im Docker-Image); die Toolbox ist entkoppelt und bleibt unberührt.

## Aufgabe
1. **Lokale Umgebung**: `brew install openjdk@25` (Freigabe des Menschen liegt vor; koexistiert mit openjdk@21); Build-Wrapper/`JAVA_HOME` auf `/opt/homebrew/opt/openjdk@25/libexec/openjdk.jdk/Contents/Home`; `infra/toolchains.xml.example` auf JDK 25.
2. **pom.xml Kern**: Parent `spring-boot-starter-parent` 3.5.0 → neueste 3.5.x (≥ 3.5.6); `<java.version>25</java.version>`; Enforcer `requireJavaVersion` `[21,22)` → `[25,26)` inkl. Kommentar.
3. **pom.xml Toolchain-Bumps** (Class-File 69): Error Prone 2.36.0 → ≥ 2.42 (JDK-25-javac; `-J--add-exports` prüfen); SpotBugs 4.8.6 → 4.9.x; pmd-core/pmd-java via Plugin-Dependencies → ≥ 7.16; PIT 1.17.0 → neueste; ArchUnit 1.4.0 → neueste 1.4.x; JaCoCo 0.8.15 verifizieren (25-Support ab 0.8.14), nur bei Bedarf bumpen. **google-java-format 1.25.2 möglichst pinnen** (Bump kann 229 Dateien neu formatieren); nur bumpen wenn auf JDK 25 nicht lauffähig — dann Reformat als separater, reiner Format-Commit.
4. **Build grün ziehen**: neue `-Xlint`-Warnungen und neue Error-Prone-Checks beheben (nicht unterdrücken; systematisches Rauschen einer Kategorie nur als dokumentierte Einzel-Ausnahme nach dem `-serial`-Muster aus #0076). Iterativ: `mvn -Dskip.frontend=true verify` und `mvn -Ppit -Dskip.frontend=true test`.
5. **Container**: `Dockerfile` Stage 2 `maven:3.9-eclipse-temurin-21` → `maven:3.9-eclipse-temurin-25`; Stage 3 `eclipse-temurin:21-jre` → `eclipse-temurin:25-jre`. Node-Stage und docker-compose-Services unverändert.
6. **CI**: `.github/workflows/ci.yml` `java-version: '21'` → `'25'`.
7. **Doku/Memory**: `CLAUDE-java.md` §1 (Version, brew-Pfad, Enforcer-Hinweis) — koordiniert mit #0079 (Versionsstrategie: „Java 25 erreicht, Boot 4.x offen"); Memory-Eintrag zur Build-Umgebung (JAVA_HOME) aktualisieren.

## Akzeptanzkriterium
