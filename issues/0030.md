## Kontext
Der sichtbare Produktname soll „kanban-kit" heißen (in Anlehnung an das claude-workflow-kit), nicht „manban". Betroffen ist ausschließlich die für Nutzer sichtbare Wortmarke/der Titel — Code-Bezeichner bleiben unverändert.

## Aufgabe
Sichtbaren Namen an diesen Stellen von „manban" auf „kanban-kit" ändern:
- `frontend/src/components/AppShell.tsx` (Zeile ~250) — Wortmarke in der AppBar.
- `frontend/src/components/AuthCard.tsx` (Zeile ~11) — Marke auf den Auth-Screens.
- `frontend/index.html` — `<title>`.
- `frontend/src/components/AppShell.test.tsx` — Assertion `getByText('manban')` auf `'kanban-kit'` nachziehen.

Ausdrücklich NICHT ändern (Code-Bezeichner, keine Nutzeranzeige):
- Paketname `manban-frontend` (package.json), Java-Package `org.mwolff.manban.*`, Env-Variablen `MANBAN_*`, DB/Cookie-Namen, der localStorage-Prefix `manban.listFilters.*` (Umbenennen würde gespeicherte Filter verwerfen).

## Akzeptanzkriterium
- `npm test` + `npm run build` grün (angepasste AppShell-Assertion).
- In der laufenden App zeigen AppBar, Auth-Screens und der Browser-Tab „kanban-kit"; keine sichtbare „manban"-Beschriftung mehr.
- `grep -rn "manban" frontend/src frontend/index.html` liefert nur noch Code-Bezeichner (Kommentare/Keys), keine sichtbaren Labels.
