## Kontext
Automatisierung/CLI-Zugang ohne interaktives Login (aus Toolbox bewährt).

## Aufgabe
- `common/token`-Crypto portieren (`TokenCryptoPort`, `Sha256TokenCryptoAdapter`, Format `tk_<64hex>`).
- `kanban_access_token`-Domain + Use-Cases (create/list/revoke/resolve, Plaintext nur einmal).
- `POST/GET/DELETE /api/access-tokens` (nur per Cookie-Login verwaltbar, nicht per PAT selbst — Least Privilege).
- Auth-Filter für Header `X-Kanban-Token`, additiv neben der Cookie-Auth, baut synthetische Authentication mit `user_id`.

## Akzeptanzkriterium
- Token anlegen liefert Plaintext einmalig.
- Request mit gültigem `X-Kanban-Token` authentifiziert.
- Widerrufener Token → 401.
- Token-Verwaltung per PAT verweigert.
- `mvn verify` grün.
