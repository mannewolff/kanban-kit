## Kontext
[CLAUDE-react.md](CLAUDE-react.md) → TypeScript / Verbotene Muster: „Keine Non-Null-Assertions (`!`) aus Bequemlichkeit." Zwei Stellen setzen `!`, obwohl der Wert kurz zuvor geprüft wurde — statt den geprüften Wert in eine lokale Konstante zu binden, wird TypeScript per Assertion überstimmt.

## Aufgabe
- `frontend/src/pages/BoardListPage.tsx:216` (`cardsApi.move(rowDrag!, …)`): geprüften Wert vorher binden, z. B. `const dragId = rowDrag; if (dragId == null) return;` und dann `dragId` verwenden.
- `frontend/src/components/BoardView.tsx:261` (`cleanupDaysRemaining(card.movedToDoneAt!, …)`): analog per Guard/`const` ohne `!`.

## Akzeptanzkriterium
- Keine der beiden `!`-Assertions mehr; Nullwert wird über echten Guard/`const` behandelt.
- `tsc --noEmit` sauber, `vitest run` grün.

Keine.
