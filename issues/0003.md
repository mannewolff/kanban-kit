## Kontext
Self-Service-Signup ist die Eintrittstür. SMTP nötig (der Punkt, der in der Toolbox #262 blockierte).

## Aufgabe
- `app_user` anlegen, Passwort-Hashing Argon2id (Spring Security `Argon2PasswordEncoder`).
- `POST /api/auth/register` (E-Mail + Passwort + Name), Verifikations-Token in `email_verification_token`.
- Versand per SMTP (`JavaMailSender`, konfigurierbar, in Dev Mock/Log).
- `GET /api/auth/verify?token=…`.
- Neu registrierte User: `email_verified=false`, kein Login bis verifiziert.

## Akzeptanzkriterium
- Registrierung legt User an, sendet Verifikations-Mail (Dev: Log).
- Verify setzt `email_verified=true`.
- Doppelte E-Mail → 409.
- Tests für Happy Path + abgelaufener/ungültiger Token.
- `mvn verify` grün.

### Hinweise
- Passwort-Hashing bewusst über spring-security-crypto (Argon2id) OHNE den vollen Web-Security-Starter — sonst Default-401 auf allen Endpunkten. SecurityConfig, Login/Session und Default-Deny für /api/** kommen mit A2; dann werden /api/auth/register + /verify explizit als public freigegeben.
- 'Kein Login bis verifiziert' ist in A1 durch email_verified=false vorbereitet; die Durchsetzung passiert im Login (A2).
- Mail-Versand default AUS (manban.mail.enabled=false) → Link wird geloggt. Für echten Versand MANBAN_MAIL_ENABLED=true + spring.mail.* setzen.
- BouncyCastle-Version gepinnt (Spring Boot managt bcprov-jdk18on nicht).
