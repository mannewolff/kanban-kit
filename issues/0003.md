## Kontext
Kern der eigenen Auth. Ersetzt den Keycloak-Resource-Server.

## Aufgabe
- `POST /api/auth/login` (E-Mail + Passwort), Session als HttpOnly-Secure-SameSite-Cookie (bevorzugt) mit serverseitig prüfbarem Token.
- `POST /api/auth/logout`.
- `GET /api/me` (Claims + Plattform-Rolle + Projekt-Mitgliedschaften).
- `SecurityConfig`: stateless bzgl. Server-Session-Store oder signiertes Cookie, CSRF-Strategie passend zum Cookie-Ansatz, Default-Deny für `/api/**`.
- 2FA-Vorbereitung: Login-Flow so strukturieren, dass ein optionaler zweiter Faktor (TOTP/WebAuthn) später eingehängt werden kann (z. B. Login liefert bei aktivem MFA ein Challenge-Ergebnis statt Session).

## Akzeptanzkriterium
- Login mit korrekten Daten setzt Cookie, `GET /api/me` liefert den User.
- Falsche Daten → 401; unverifizierte User → 403.
- Logout invalidiert.
- Tests inkl. Security-Slice.
- `mvn verify` grün.
