### Tests und Checks
- `mvn verify` → BUILD SUCCESS: 1 Unit + 27 Integrationstests (28 gesamt).
- RbacMatrixIT: erschöpfende Matrix (jede Rolle × jedes Recht) gegen den Seed — VIEWER keine, MEMBER {CARD_*, COMMENT_CREATE, ATTACHMENT_UPLOAD}, ADMIN + {COLUMN_EDIT, BOARD_*, MEMBER_*}, OWNER alle. Plus Endpunkt: ADMIN darf Projekt nicht umbenennen (403), OWNER schon (200).

### Hinweise
- Der PermissionChecker ist die eine Stelle für 'hat User Recht X im Projekt Y?' und wird von den folgenden Issues (P3 Mitgliederverwaltung, B1 Board/Spalten, B2–B5 Karten/Kommentare/Anhänge) für ihre verändernden Aktionen genutzt.
- Matrix ist tabellengestützt (role_permission) — ein späterer Admin-Editor bliebe reines UI, kein Code-Umbau.
- Nichtmitglied → 404 (kein Existenz-Leak), Mitglied ohne Recht → 403 (konsistent mit P1).
