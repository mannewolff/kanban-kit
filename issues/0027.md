## Kontext
Das aktuelle `CardDetailModal` bearbeitet nur die Beschreibung separat; Titel, Epic-Zuordnung und Abhängigkeiten sind über verstreute Bedienelemente verteilt. Die Toolbox (`../java/frontend/src/pages/kanban/KanbanDetailModal.tsx`) löst das als ein einziges Edit-Formular mit Lese-/Bearbeiten-Umschaltung. Ziel: wortgetreuer Port, nur Datenanbindung angepasst (dynamische Board-Spalten statt Status-Enum). Backend `CardService.update` setzt bisher keinen `parentId`.

## Aufgabe
- Backend: `CardController.UpdateCardRequest` + `CardService.update` um `parentId` (nullable) erweitern — Titel/Beschreibung/Dependencies/Epic-Zuordnung in EINEM PUT. Validierung wie `assignParent` (nur type=CARD, Epic muss zum Board gehören). Der bestehende `/api/cards/{id}/parent`-Endpunkt bleibt. Backend-IT: update mit parentId ordnet zu/löst.
- `frontend/src/api/cards.ts`: `update` um `parentId` erweitern.
- `frontend/src/components/CardDetailModal.tsx` neu nach `KanbanDetailModal.tsx`:
  - Header: Status-Chip (`statusColors(column.name)`) + `#num` + Titel + BEARBEITEN-Button.
  - Lesemodus: gerenderte Markdown-Beschreibung in umrandeter Box (GitHub-artiges `descriptionSx`), "Abhängig von: #.."-Zeile, Anhänge, Kommentare.
  - Edit-Modus als ein Formular: Titel + Markdown-Beschreibung + Epic-Dropdown (native) + "Abhängig von (Nummern, kommagetrennt)" (`parseDependencyInput` portieren) — Speichern/Abbrechen in DialogActions; ein PUT über `cardsApi.update(id, title, body, deps, shortcode?, parentId)`.
  - Bei `type==='EPIC'`: Kürzel-Feld + Kinderliste (bestehendes Verhalten erhalten), kein Epic-Dropdown.
  - Spalten-/Done-Bezug dynamisch (kein Enum). Task-Checkbox-Toggling NICHT in diesem Issue.
- Tests: `parseDependencyInput` (rein), CardDetailModal Lese-/Edit-Modus + Speichern ruft update mit allen Feldern; EpicsPage-/BoardPage-Nutzung bleibt grün.

## Akzeptanzkriterium
- `npm test` + `npm run build` grün; `mvn verify` grün (update-parentId-IT).
- Detail öffnet im Lesemodus (gerendertes Markdown), BEARBEITEN zeigt EIN Formular mit Titel + Beschreibung + Epic + Abhängigkeiten; Speichern persistiert alles in einem Request.
