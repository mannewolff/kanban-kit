## Kontext
[CLAUDE-react.md](CLAUDE-react.md) → Hooks: „Asynchrone Effekte müssen Race-Conditions verhindern (Cancellation-Flag oder `AbortController`)." Mehrere Effekte mit Dependency `[id]` feuern bei Board-/Projektwechsel `.then(setState)`-Fetches ohne Abbruch — eine spät auflösende Antwort für die alte `id` kann den State der neuen überschreiben. `BoardPage.tsx:38` und `AppShell.tsx:86` machen es mit `let active = true` bereits vor.

## Aufgabe
- Cancellation-Flag (`let active = true` + Cleanup `return () => { active = false }`, `setState` nur bei `active`) oder `AbortController` ergänzen in:
  - `frontend/src/pages/BoardListPage.tsx:91-108`
  - `frontend/src/pages/EpicsPage.tsx:45-49`
  - `frontend/src/pages/ProjectBoardsPage.tsx:34-51`
  - `frontend/src/pages/ProjectMembersPage.tsx:39-46`
- Analog zum bestehenden Muster in `BoardPage.tsx`.
- Test ergänzen, der einen späten Response nach `id`-Wechsel verwirft (kein State-Update).

## Akzeptanzkriterium
- Jeder genannte Effekt räumt bei `id`-Wechsel/Unmount auf und ignoriert veraltete Antworten.
- `vitest run` grün, `tsc --noEmit` sauber.
