## Kontext
Das Backend (#0101) unterscheidet jetzt direkte Mitgliedschaft (`added`) von Einladung (`invited`) und lehnt nicht freigegebene Nutzer mit 422 ab. Die UI soll das rückmelden, damit der Anwender versteht, was passiert ist.

## Aufgabe
- Einladen-Dialog / Members-View im `frontend/`: Response `added|invited` auswerten → passende Erfolgsmeldung („… wurde hinzugefügt" vs. „Einladung verschickt").
- 422 `UserNotApprovedException` → verständliche Fehlermeldung („Nutzer ist noch nicht vom Admin freigegeben").
- Vitest-Tests.

## Akzeptanzkriterium
- Test: `added` → „hinzugefügt"-Meldung; `invited` → „Einladung"-Meldung; 422 → Freigabe-Hinweis.
- `npm run build` + `npm run lint` + `npm test` grün.
