## Kontext
Registriert und verifiziert sich ein neuer Nutzer, sollen alle Plattform-Admins per Mail informiert werden, damit sie ihn freigeben können (C3: Auslösung nach Verifikation; C4: an alle Admins). Baut auf dem Approval-Feld aus #0097 auf.

## Aufgabe
- Neue Repo-Query `findByPlatformRole(PlatformRole)` in `AppUserRepository` (Port) + `AppUserJpaRepository` + Adapter.
- Neuer Mailer-Port `AdminNotificationMailer` (`sendNewUserPendingApproval(adminEmail, newUserEmail, newUserDisplayName)`) + `JavaMailAdminNotificationMailer` nach bestehendem Muster (`manban.mail.enabled`-Log-Fallback, wie [JavaMailVerificationMailer](src/main/java/org/mwolff/manban/auth/infrastructure/mail/JavaMailVerificationMailer.java)).
- Auslösung in `VerifyEmailService.verify(...)` ([auth/application/VerifyEmailService.java:45](src/main/java/org/mwolff/manban/auth/application/VerifyEmailService.java)) **nach** `withEmailVerified(true)`: nur wenn User **nicht** bereits freigegeben (`approvedAt == null`) → an alle Admins (`findByPlatformRole(ADMIN)`) senden.
- TDD zuerst.

## Akzeptanzkriterium
- IT (Mailpit): Verifikation eines pending Users → je Admin eine Mail; bei bereits freigegebenem User (z. B. eingeladen, #0099) **keine** Mail.
- `manban.mail.enabled=false` → nur Log-Ausgabe, kein Versand.
- `mvn verify` grün.
