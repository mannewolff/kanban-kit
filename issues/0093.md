## Kontext
Für die Veröffentlichung wird das GitHub-Repository auf `mannewolff/kanban-kit` umbenannt (#0092). Danach zeigen die Repo-/Klon-URLs im Repo noch auf `mannewolff/manban`. GitHub leitet alte URLs zwar automatisch um, aber für ein sauberes öffentliches Repo sollen die Referenzen stimmen. Der **interne Codename** manban (Java-Package `org.mwolff.manban`, Env-Prefix `MANBAN_*`, Docker-Service `manban-api`, DB, `frontend/package.json` name) bleibt **bewusst unangetastet** — keine Verhaltensänderung.

## Aufgabe
- Die drei aktiven `mannewolff/manban`-URL-Referenzen auf `mannewolff/kanban-kit` umstellen:
  - `README.md:23` — `git clone https://github.com/mannewolff/manban.git`
  - `docs-site/.vitepress/config.ts:63` — GitHub-Social-Link
  - `docs/deployment-hostinger.md:44` — `git clone -b production https://github.com/mannewolff/manban.git`
- **Nicht** anfassen: interne `manban`-Vorkommen (Package, `MANBAN_*`, `manban-api`, `manban.jar`, DB-Name, `manban-frontend`, localStorage-Keys) und die `issues/`-Historie/`issues/archive/`.

## Akzeptanzkriterium
- `grep -rn "mannewolff/manban" .` (ohne `node_modules`, `target`, `issues/`) liefert **keine** Treffer mehr.
- `cd frontend && npm run build` grün (keine Verhaltensänderung; reine String-/Doku-Anpassung).
