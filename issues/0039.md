## Kontext
Anforderung: klären/sichtbar machen, welche Rolle welche Rechte hat. Die Projekt-RBAC-Matrix (OWNER/ADMIN/MEMBER/VIEWER → Permissions) ist im Backend geseedet (`role_permission`), aber für Nutzer nirgends sichtbar. Zusätzlich die neue Plattform-Rolle (USER vs. ADMIN = Super-User) erklären.

## Aufgabe
- Eine statische, gut lesbare **Rechte-Übersicht** im Frontend (z. B. `pages/RolesPage.tsx`, Route `/roles`, Sidebar-Eintrag oder Link aus der Mitglieder-/Admin-Seite):
  - Tabelle Projekt-Rollen × wichtigste Rechte (Board/Karten/Kommentare/Anhänge/Mitglieder verwalten …) — inhaltlich konsistent zur Seed-Matrix.
  - Kurzer Abschnitt Plattform-Rollen: USER (nur eigene/Mitglieds-Projekte) vs. ADMIN (Vollzugriff auf alle Projekte + Nutzerverwaltung).
- Quelle der Wahrheit: die Rechte-Beschreibung soll mit der tatsächlichen Matrix übereinstimmen; wenn ein einfacher Endpunkt (`GET /api/roles` o. ä.) den Aufwand rechtfertigt, kann die Übersicht datengetrieben sein — sonst als gepflegte statische Tabelle mit Verweis auf die Seed-Migration.
- Tests: Seite rendert die Rollen und mindestens die Kern-Rechte; Build grün.

## Akzeptanzkriterium
- `npm test` + `npm run build` grün.
- Nutzer können eine verständliche Übersicht "welche Rolle darf was" aufrufen (Projekt-Rollen + Plattform-Admin).
